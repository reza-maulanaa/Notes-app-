# Product

## Register

product

## Users

Personal use — the person building and running this app. Context: desktop browser, writing environment, probably have multiple tabs open. Primary task is capturing and retrieving notes fast.

## Product Purpose

A personal notes app for quick capture and retrieval. Success looks like: note is written, saved, and found again without friction. Zero ceremony.

## Brand Personality

Sharp, lean, present. The interface disappears; only the content stays.

## Anti-references

- Google Keep: colorful card grid, playful label colors, too much visual noise for plain text notes
- Generic SaaS cream backgrounds with AI-default warm-neutral surfaces

## Design Principles

1. **Content first** — the note is the product; the chrome is the packaging. Minimize chrome.
2. **One decision per screen** — each page has one primary action. Everything else is secondary.
3. **No decoration without function** — color, border, shadow only when they convey meaning or aid orientation.
4. **Earned familiarity** — borrow conventions from Linear, Notion, and system UIs. Don't reinvent affordances.
5. **Light is not neutral** — pure white with one confident accent reads sharp and intentional; warm-tinted surfaces read as AI default.

## Accessibility & Inclusion

WCAG AA minimum. Reduced motion respected. Keyboard-navigable forms.
